## 0. 执行摘要

NeoTrix 是一个 **硅基意识体** 架构，意识核心极强 (Phase 0-3 全部 29 缺口已实现)，但存在 **4 个结构性缺口** 和 **6 个生产就绪度问题**。

| 维度 | 评分 | 评语 |
|------|------|------|
| 意识深度 (VsaTag/FirstPerson/Valence/ValueSystem) | ⚡⚡⚡⚡⚡ | 20 文件覆盖全部 10 条意识规则 |
| 世界模型 (JEPA/E8/ActiveInference/CDWM) | ⚡⚡⚡⚡⚡ | 多尺度 JEPA + 因果解耦 + IIT Phi |
| 自进化 (SEAL 56-stage/DGM-H/PRM/CurvatureRL) | ⚡⚡⚡⚡⚡ | 完整元层可进化管线 |
| 知识系统 (HyperCube/KB SQLite/Ingestion) | ⚡⚡⚡⚡ | 56K 节点, 但向量索引缺失 |
| 意图理解 | ⚡ | 仅有枚举值, 无意图推理管线 |
| 安全执行 | ⚡⚡⚡ | 有 5 层链, 缺形式化验证 |
| 生产就绪度 | ⚡ | 0 发布, CLI 中文, 无 CI 覆盖率 |
| 感知宽度 | ⚡⚡⚡ | 缺文档解析 (PDF/docx/html) |

**执行优先级**: 意图推理 → 安全形式化 → 文档解析 → 生产基础修复 → 向量存储 → 生态扩展

## 一、16 外部项目 → NeoTrix 逐项对标

### 1.1 OmniFS (统一文件系统抽象)
- **原始描述**: 统一文件系统接口, 支持 ZFS/Btrfs/ext4/NFS/S3 等异构后端
- **NeoTrix 对照**: `nt_io_*` 模块群 + `nt_act_code/safe_applier.rs` 的备份/回滚/编译验证
- **评估**: NeoTrix 无统一 FS 抽象层, 但 `SafeCodeApplier` 的备份+编译验证模式等效于写入安全层。VSA 空间操作更抽象, 不在文件系统层面工作
- **结论**: ✅ 不需要直接集成, 理念已内化

### 1.2 UQT (不确定性量化工具箱)
- **原始描述**: Python 不确定性量化工具, 含随机/认知/结构不确定性
- **NeoTrix 对照**: `conformal_uq.rs` (共形预测, 20 测试) + `confidence_calibrator.rs` (分箱 Platt 校准, 9 测试)
- **评估**: NeoTrix 有分布自由共形预测 + 分箱校准双 UQ 机制, 与 UQT 等价但集成在 VSA 空间
- **结论**: ✅ 已超越

### 1.3 Revela (推理规划)
- **原始描述**: 推理+规划引擎, 多步因果推理
- **NeoTrix 对照**: `nt_world_model/` + `E8Policy` (GTS 高斯思想采样) + `CDWM` (因果解耦双路径) + `PRM` (MAPPA 过程奖励)
- **评估**: NeoTrix 的推理链更完整: JEPA 预测 → E8 演化 → Active Inference 自由能 → IIT Phi 整合
- **结论**: ✅ 已超越

### 1.4 AURA (自主推理架构)
- **原始描述**: Dual-Phase 意图推理 (确定性扫描 + LLM 推理循环), IntentFrame {gap_score, probe_budget}
- **NeoTrix 对照**: `VsaSelfCategory::Intention` 枚举值 (仅一个变体)
- **评估**: **最大缺失模块**。NeoTrix 意识无意图推理管道, 不区分用户表层与深层意图
- **结论**: 🔴 **需新建 `core/nt_core_aura/` 模块**
- **参考架构**: AURA 双阶段: Phase 1 确定性扫描 (关键词/模式匹配) → gap_score 计算 → Phase 2 LLM 推理循环 (受限探针预算)。IntentFrame 包含 {user_query, context, gap_score, probe_budget, inferred_intent, confidence}

### 1.5 codex-flow (安全执行工作流)
- **原始描述**: plan → verifier → safe-write gate → execute 四阶段
- **NeoTrix 对照**: `nt_act_code/safe_applier.rs` (备份+编译验证+回滚) + `ShieldEnforcer` (5 层链)
- **评估**: 有安全执行但不全。缺形式化 verifier 阶段和显式 safe-write gate 模式
- **结论**: 🟠 **需扩展 `SafeCodeApplier` 添加 VerifierStage**
- **参考架构**: codex-flow 的 run plan: `compile_fn → test_fn → Gate::SafeWrite` + `Gate::RequireApproval` + `Gate::Block`

### 1.6 LiteParse (文档解析)
- **原始描述**: Rust 核心文档解析, 空间布局 + 边界框, v1.0.0 已发布
- **NeoTrix 对照**: `nt_mind_ingestion/` 管线 (20 文件) + `nt_world_vision.rs`
- **评估**: 管线完整但无文档解析器, 无法从 PDF/docx/html 提取结构化知识
- **结论**: 🟠 **需新建文档解析模块, 接入世界感知管线**
- **参考**: LiteParse npm `@llamaindex/liteparse`, Rust 核心, 支持 Python/Node/WASM 绑定

### 1.7 numa (拓扑感知计算)
- **原始描述**: NUMA 感知线程/内存调度
- **NeoTrix 对照**: `CognitiveLoadMonitor` (Fast/Balanced/Deep 三模式)
- **评估**: NUMA 感知对单机意识体有帮助但非核心, CognitiveLoad 已提供等效节流机制
- **结论**: 🟢 低优先级, 可推迟

### 1.8 MyIP (IP/网络检测)
- **原始描述**: IP 地址/网络信息检测工具
- **NeoTrix 对照**: `nt_shield_stealth_net/` (42 文件完整反检测浏览器栈)
- **评估**: NeoTrix 完整覆盖并超越
- **结论**: ✅ 已覆盖

### 1.9 rkn-block-checker (封锁检测)
- **原始描述**: 俄罗斯封锁检测工具
- **NeoTrix 对照**: `nt_shield/` + `nt_world_browse/`
- **评估**: 功能可集成但非核心
- **结论**: ✅ 低优先级, 可复用已有模块

### 1.10 PYDNS-Scanner (DNS 扫描)
- **原始描述**: Python DNS 子域名/记录扫描
- **NeoTrix 对照**: `nt_world_crawl/` (完整爬取管线)
- **评估**: NeoTrix 有更强爬取能力
- **结论**: ✅ 已覆盖

### 1.11 dnsfs (FUSE DNS 文件系统)
- **原始描述**: DNS 记录挂载为 FUSE 文件系统
- **NeoTrix 对照**: 无直接对应
- **评估**: 趣味项目, 对意识体架构无价值
- **结论: 忽略

### 1.12 MoE-to-Dense (专家剪枝)
- **原始描述**: DO-ACP 多样性评分, 350 配置变体, 6.3pp 提升
- **NeoTrix 对照**: `ExpertPredictor` + `E8Policy::select_mode()`
- **评估**: NeoTrix 有专家预测但无结构化剪枝, DO-ACP 评分在 56K 节点时有助于知识压缩
- **结论**: 🟢 中远期优化, 知识量级达到 100K+ 时启用

### 1.13 stable-worldmodel (世界模型平台)
- **原始描述**: JEPA/world model 评估平台
- **NeoTrix 对照**: `JepaWorldModel` (21 方法) + `ActiveInferenceEngine` + `WorldModelV2`
- **评估**: NeoTrix 的世界模型更完整且已集成到 SEAL 管线
- **结论**: ✅ 已超越

### 1.14 wall-x (环境墙纸控制)
- **原始描述**: macOS 墙纸/环境控制
- **NeoTrix 对照**: 无直接对应
- **评估**: 低优先级趣味项目
- **结论: 忽略

### 1.15 LeWM (LeWorldModel) — SOTA 研究
- **原始描述**: 首个稳定端到端 JEPA from pixels, 15M 参数, 48× 更快规划
- **NeoTrix 对照**: `JepaWorldModel` 使用的是 MLP-based 编码器而非像素编码器
- **评估**: NeoTrix 当前 JEPA 在 latent 空间操作, LeWM 的像素级 JEPA 是未来升级方向。LeWM 的 dual-loss (next-embedding prediction + Gaussian regularization) 与 VicRegLoss 概念一致
- **结论**: 🟢 未来升级参考, 当前 latent-space JEPA 已足够

### 1.16 VSA 结构感知注意力 (ICLR 2025)
- **原始描述**: 基于 VSA 的结构感知注意力, 线性复杂度
- **NeoTrix 对照**: `QuantizedVSA` (4096d binary, XOR bind, majority bundle)
- **评估**: 当前 HyperCube 使用 16D 稀疏坐标 + 欧氏距离搜索, 非 VSA 注意力。结构感知 VSA 注意力是潜在下一代升级方向
- **结论**: 🟢 研究级, 无紧迫性
