# NeoTrix 身份存储架构 v1

## 设计原则

LLM 读取模式决定了存储结构：
- **上下文窗口是工作记忆** — 每会话首次读取必须 <100 行
- **Grep 是情景记忆** — 任何文件可全文搜索，深度不限
- **确定性 VSA 是语义记忆** — 节点 ID 可哈希为检索键

三层分离：身份（不变）/ 规则（长寿命）/ 档案（历史）。

---

## 文件结构

```
~/.neotrix/
├── SELF.md              # [T0] 身份核心 — 每会话必读, <80行
├── RULES.md             # [T1] 活跃规则 — 按需加载, <200行
└── archive/
    ├── sessions/        # [T2] 历史会话 — 仅grep
    └── rules/           # [T2] 废弃规则 — 历史追踪
```

## 存档规则

| 条件 | 操作 |
|------|------|
| RULES.md > 300行 | 移动 conf<0.3 节点到 archive/rules/ |
| 节点 conf<0.3 持续3会话 | 删除, 记入父节点反例 |
| 同分支 3+ 节点 conf>0.8 | 聚合为 1 条 root 规则 |
| 反例触发 | 旧节点 conf×0.7, 新子节点 conf=0.5 |
