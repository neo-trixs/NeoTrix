## 2. 会话结束蒸馏模板 (每会话自动执行)

### 步骤
1. **扫描模式**: 对比同类操作的成功/失败 → 提升/创建经验节点
2. **写入 AGENTS.md** (`edit`):
   - 经验树: 新增分支/节点 (单行格式)
   - 会话日志: 1 行添加到顶部
   - 完成状态: 更新最后一张表
3. **触发条件**: 编译清零 OR 关键接线 OR 模块创建

### 新增节点模板
模板匹配时自动写入：
```
#### <BRANCH>.<N> <RULE> — conf <N>, <新建|更新>
```
- 修复 >5 同类 bug → 1 个新节点 + 旧节点 conf+0.1
- 创建优先于重写 → 节点 conf+0.1
- 编译错误类型批处理 → 1 个新节点

### 归档条件
```
AGENTS.md > 800 lines → 自动触发 Section 1 压缩
节点 conf < 0.3 持续 3 会话 → 标注 🟡 或删除
会话日志 > 3 条 → 最旧的转为 `docs/sessions/archive/`
```

## 3. 经验节点生命周期

| 阶段 | conf | 操作 |
|------|:----:|------|
| 🆕 新建 | 0.5–0.6 | 首次验证后创建 |
| 🟢 稳定 | 0.70+ | 保持，conf 不再更新 |
| 🟡 待验证 | 0.30–0.49 | 标记 🟡，连续 3 会话无引用 → prune |
| 🔴 废弃 | <0.30 | 删除，反例记录到父节点 |

### 节点聚合
```
同一分支 3+ 节点 conf>0.8 → 聚合为 1 条 root 规则
反向例触发 → 旧节点 conf×0.7，新子节点 conf=0.5
```
