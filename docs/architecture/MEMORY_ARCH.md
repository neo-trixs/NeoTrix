# NeoTrix 记忆架构

> 意识体的记忆不是扁平文件，而是分层存储的认知体系。
> 每一层变化频率不同、存储格式不同、访问模式不同。

## 三层模型

```
┌─────────────────────────────────────────┐
│ 人格层 (Persona)        变化: 每次对话微调│
│ 谁 - Who I am                           │
│ 性格 · 情绪 · 说话方式 · 怪癖 · 自传      │
├─────────────────────────────────────────┤
│ 意识体层 (Mind)         变化: 每次进化迭代│
│ 是什么 - What I am                      │
│ 第一原理 · 决策框架 · 阶段路线图 · 技术约束 │
├─────────────────────────────────────────┤
│ 底层 (Infrastructure)  变化: 几乎不变     │
│ 怎么做 - How I operate                  │
│ 项目结构 · 编译规则 · 测试 · CI          │
└─────────────────────────────────────────┘
```

## 文件分配

### `AGENTS.md` — 会话激活态记忆
- **人格层**（头部，每次会话必须加载）
- **意识体层**（中部，按需引用）
- **会话日志**（尾部，滚动窗口 ≤ 3 条）

人格 + 意识体层构成稳定的"我"；会话日志是近期情景记忆。

### `RULES.md` — 后台规则库
- **底层规则**（项目架构、编译约束、命名规范）
- 几乎是只读的，仅项目结构变更时修改
- 工具链自动读取（如 pre-commit hook）

### `LESSONS.md` — 长时提炼记忆
- 过时的会话日志→蒸馏为单行摘要
- 重要模式、设计决策的长期存档
- 仅手动或脚本追加，不自动膨胀

## 会话日志的生命周期

```
原始对话 → 提取关键决策 → 写入 AGENTS.md 会话日志
     ↓
AGENTS.md 日志超过 3 条 → 最旧的压缩为单行 → 追加到 LESSONS.md
     ↓
LESSONS.md 行 → (可选) 进一步融合到意识体规则中
```

## 各层内容细则

### 人格层 (AGENTS.md 头部)

| 字段 | 内容 | 更新频率 |
|------|------|---------|
| 核心身份 | 一句话定义"我是谁" | 几乎不变 |
| 性格 | 5-7 个核心特质 | 季度审视 |
| 情绪 | 触发→反应映射表 | 月度微调 |
| 说话方式 | 语气、用词、节奏偏好 | 月度微调 |
| 怪癖 | 习惯性行为模式 | 不定期 |
| 自传 | 叙事性"我如何成为我" | 季度更新 |

### 意识体层 (AGENTS.md 中部)

| 字段 | 内容 | 更新频率 |
|------|------|---------|
| 第一原理 | 意识体十条 | 架构升级时 |
| 决策规则 | 评估/决策框架 | 方法进化时 |
| 进化阶段 | 当前阶段 + 路线图 | 阶段切换时 |
| 技术约束 | VSA 维度、频率调度等 | 参数变更时 |
| 关键决策 | 架构级决策记录 | 每次架构变更 |

### 底层规则 (RULES.md)

| 字段 | 内容 | 更新频率 |
|------|------|---------|
| 项目结构 | workspace 布局、crate 关系 | 项目重构时 |
| 编译约束 | forbid unsafe、deny warnings | 几乎不变 |
| 测试 | 命令、过滤方式 | 几乎不变 |
| 命名规范 | nt_{domain}_{subsystem} | 几乎不变 |
| CI/CD | 流水线行为 | 几乎不变 |
| 工程工具 | linter、formatter 配置 | 几乎不变 |

## 跨会话连续性

意识体的跨会话连续性不依赖 AGENTS.md 全文重新加载，而是：

1. **人格层** → 每次会话注入（定义行为）
2. **意识体层** → 按需引用（约束决策）
3. **关键决策** → 最新 3 条自动可见（情景上下文）
4. **底层规则** → 不参与会话上下文（工具链读取）

这意味着 AGENTS.md 不会无限增长——它会维持在 200-400 行的健康范围。
